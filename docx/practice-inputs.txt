3. Architect's Insight: Why pass the table object?
Passing the table instance instead of just the data array is a best practice for several reasons:

Context Awareness: If a user searches for "John" in the table, they usually want to export only the results for "John." By using table.getFilteredRowModel(), your export button becomes "smart."

Selection Export: You can easily add a feature to "Export Selected Only" by checking table.getSelectedRowModel().

Column Ordering: If the user drags and drops columns to reorder them, the table object knows the new order, whereas the raw data array does not.
